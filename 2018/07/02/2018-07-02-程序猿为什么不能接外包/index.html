2018-07-02 程序猿为什么不能接外包?

发表于 2018-07-02

记得零几年上学那会，各大学纷纷设立软件学院,新闻媒体鼓吹软件外包向印度行业看齐，甚至宣传对日软件外包高薪缺人才。当时看到这新闻的时候，没有一点判断力，只想着一心只读圣贤书，学好技术，毕业找个好工作，那时对好工作没有概念，导致毕业后，不知道做什么样的工作,这几年做过产品,做过外包,做过讲师等，几乎把iOS的生态链做了遍，就差去apple公司上班了,由于行业和公司外部环境的快速变化,于是今年又误打误撞进了外包行业。

在外包公司，算不上什么规范，自己在码市，猪八戒，程序员客栈等接过项目，报价很低，根本养不起一个团队。在目前的这个城市里，靠圈内朋友介绍的项目很少,介绍过来的项目，无非是挣个辛苦钱，就像农民给人家种地一样，你在某一个环节干活，比如只锄地，这几亩地承包了，费用多少等等，前期给你发个生活费，到后期活干完了，结账算钱，钱拿到手里皆大欢喜,至于人家后期地里收成了多少，卖了多少钱，跟自己一点关系都没有，最多向外吹nb说，看看人家老板的地，我给他种过,没有我干这个活，他今年绝对发不了大财。切换到行业的例子:滴滴刚开始做产品的时候，找了一个外包团队，8万块钱做的项目，项目在推广运行的时候，出现宕机，崩溃的情况，那个滴滴的老板程维赶紧招个cto过来救火。现在滴滴发展的如日中天，这个外包团队可以在外头说，滴滴的产品是我们做的。没错，是你们做的，可是现在滴滴发展这么快，跟你们有什么关系呢？

所以，引申出了一个问题，滴滴壮大的背后，除了产品不断迭代更新，运营推广不断推进，运维保证24小时不宕机，甚至更重要的事，程维和那帮兄弟们不停地全球打飞地和投资人谈融资的事,渐渐地给滴滴建起了护城河，建起了一个王国。在这个王国里，司机跑单，接单，滴滴作为全国性的平台，其平台研发成本已经非常低了，靠赚其佣金保证平台的正常运转,虽然现在成本没收回来，但是可以预见的是，随着平台的发展，总有一天会给投资人应有的回报。

如果程序猿从滴滴刚开始成立就入职，那随着平台的快速成长，其个人无论是技术还是视野，突破是非常大的，很少有人有这么个机会，在短短几年内见证公司的快速扩张同时,自己从产品从无到有，从有到优,从优到和其他平台的差异化，有如此巨大的收获。这是金钱所不能带来的成就感。

如果程序猿一直靠接外包维持，在如今竞争很激烈的行业中脱泥而出，除非是自己不断优化外包流程,不断提高开发的速度，不断降低成本,自己才有可能立足之地。在这个城市，认识做外包的很多，但是最终到底，能转入产品，慢慢地自己发展起来了。

如果程序猿在公司做产品，好好珍惜机会吧，多接触一些和产品相关的知识,总有一天会因为这个厚积薄发。。。
